# 用户表设计

## 最终表结构

```sql
CREATE TABLE users (
    id              BIGSERIAL PRIMARY KEY,
    username        VARCHAR(50)  NOT NULL UNIQUE,   -- 登录（唯一）
    email           VARCHAR(100) NOT NULL UNIQUE,   -- 登录/通知（唯一）
    password_hash   VARCHAR(255) NOT NULL,          -- BCrypt 加密
    nickname        VARCHAR(50),                    -- 展示昵称
    avatar_url      VARCHAR(500),                   -- 头像链接
    bio             TEXT,                           -- 个人简介
    role            VARCHAR(20)  NOT NULL DEFAULT 'USER',  -- USER / ADMIN
    status          VARCHAR(10)  NOT NULL DEFAULT 'ACTIVE', -- ACTIVE / BANNED
    created_at      TIMESTAMP    NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP    NOT NULL DEFAULT NOW()
);
```

## 设计原则

**用户表只存"身份 + 简介"，不存业务数据。**

| 保留 | 原因 |
|------|------|
| `username`, `email`, `password_hash` | 登录认证核心字段 |
| `nickname`, `avatar_url`, `bio` | 用户公开资料 |
| `role`, `status` | 权限控制和账号状态 |

## 移除的业务字段

| 移除字段 | 原因 | 后续方案 |
|----------|------|----------|
| `points_balance`(积分) | 业务数据，不属于用户身份 | 独立 `user_points` 表 |
| `ai_quota`(AI次数) | 业务数据，不属于用户身份 | 独立 `user_ai_quota` 表 |
| `api_key_encrypted`(API密钥) | 业务数据，涉及单独安全策略 | 独立 `user_api_keys` 表 |
| `api_key_provider`(API提供商) | 同上 | 同上 |

## 索引

```sql
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_username ON users(username);
```
