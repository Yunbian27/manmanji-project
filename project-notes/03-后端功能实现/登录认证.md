# 登录认证

## 技术栈

Spring Security 6 + JWT（RSA 非对称加密）+ BCrypt + Redis

## 整体流程

```
POST /api/auth/login { account, password }
  → AuthController.login()
    → AuthService.login()
      → AuthenticationManager.authenticate()
        → CustomUserDetailsService.loadUserByUsername(account)
          → 先查 username，查不到再查 email
        → BCrypt 验密码
      → JwtService.generateAccessToken() + generateRefreshToken()
      → Redis 存 refresh token
  → 返回 { accessToken, refreshToken, user }
```

## 核心组件

| 组件 | 文件 | 职责 |
|------|------|------|
| 安全配置 | `SecurityConfig.java` | 定义哪些路径公开、哪些需认证，注册过滤器 |
| JWT 工具 | `JwtService.java` | RSA 私钥签发、公钥验签 |
| JWT 过滤器 | `JwtAuthFilter.java` | 每个请求拦截，解析 JWT，设置认证状态 |
| 用户详情服务 | `CustomUserDetailsService.java` | 从 DB 查用户，返回 Spring Security 能理解的格式 |
| 认证服务 | `AuthService.java` | 登录/注册/刷新/退出的业务逻辑编排 |
| 认证控制器 | `AuthController.java` | REST 端点 |

## 双 Token 机制

| | Access Token | Refresh Token |
|---|---|---|
| 有效期 | 15 分钟 | 7 天 |
| 用途 | 访问 API | 换新的 Access Token |
| 存储 | 仅客户端 | 客户端 + Redis（jti） |
| 退出时 | 加入 Redis 黑名单 | 从 Redis 删除 |

## 无感刷新流程

```
Access Token 过期 → 返回 401
  → 前端拦截器用 Refresh Token 调 POST /api/auth/refresh
    → 后端验 Refresh Token + 重用检测
    → 签发新的一对 Token
  → 前端用新 Token 重试原请求
  → 用户无感知
```

## 安全措施

- **密码**: BCrypt 哈希存储，永不明文
- **JWT 签名**: RSA 非对称密钥，私钥在服务器，公钥可公开
- **Refresh Token 重用检测**: 旧 refresh token 一旦被替换再次使用，整个用户的所有 refresh token 作废，强制重新登录
- **Session 管理**: 无状态（STATELESS），不依赖服务器 Session
- **登出**: Access Token 加入 Redis 黑名单（TTL = 剩余有效期），Refresh Token 从 Redis 删除

## 登录方式

支持用户名和邮箱两种方式登录。`CustomUserDetailsService.loadUserByUsername()` 收到前端输入的值后：
1. 先按 `username` 列查询
2. 查不到再按 `email` 列查询
3. 都查不到则返回"用户不存在"

## API 端点

| 端点 | 方法 | 认证 | 说明 |
|------|------|------|------|
| `/api/auth/login` | POST | 否 | 登录 |
| `/api/auth/register` | POST | 否 | 注册 |
| `/api/auth/refresh` | POST | 否 | 刷新令牌 |
| `/api/auth/logout` | POST | 是 | 退出登录 |
