# 慢慢记 (ManManJi) — 项目说明文档

> **数据库迁移：** 不使用 Flyway，采用 Spring Boot `spring.sql.init` 机制（schema.sql + data.sql 直接执行）

---

## 一、产品定位

**慢慢记**——一个反内卷的技术知识平台。好记性不如烂笔头，AI 帮你快快写，读者慢慢看。

- Admin 写技术文章，沉淀知识
- 其他开发者注册账号来阅读、评论、点赞、收藏
- AI 自动向量化文章内容，访客可通过聊天窗口向知识库提问
- 积分体系 + 秒杀/兑换 AI 权益 + 热门排行

| 技术 | 版本 | 用途 |
|---|---|---|
| JDK | 21 | 运行环境 |
| Spring Boot | 4.0.1 | 主框架 |
| Spring AI | 2.0-M4 | AI 多模型接入 |
| Spring Security + JWT (RS256) | 6.x / jjwt 0.12.x | 认证授权 |
| MyBatis-Plus | 3.5.x | ORM |
| PostgreSQL + PGVector | 16 | 业务数据 + 向量存储 |
| Redis | 7.x | 缓存 / 分布式锁 / 热榜 / 签到 / Lua 秒杀 |
| Vue 3 + Nuxt 3 + Tailwind CSS | 最新 | 前端 |

## 二、数据库迁移方案（不使用 Flyway）

```yaml
spring:
  sql:
    init:
      mode: always
      schema-locations: classpath:schema.sql
      data-locations: classpath:data.sql
```

- `schema.sql` — 全部建表（`CREATE TABLE IF NOT EXISTS`，含 PGVector 扩展）
- `data.sql` — 初始数据（`INSERT ... ON CONFLICT DO NOTHING` 幂等）
- 不使用 Flyway，应用启动时自动执行

## 五、各模块业务说明

### 5.1 认证模块 (auth)

- 注册 → BCrypt 加密 → 写入 users 表 → 返回 JWT
- 登录 → 校验密码 → Access Token (15min) + Refresh Token (7d)
- JWT 使用 RS256 非对称签名（公钥验证、私钥签发）
- Refresh Token 可换新 Access Token

### 5.2 用户模块 (user)

- 个人主页：查看用户信息 + 文章列表 + 积分
- 关注/取关：toggle 幂等操作
- BYOK：用户 AES-256-GCM 加密存储自己的 OpenAI/DeepSeek Key，AI 调用时临时创建模型用完即销毁

---

### 5.3 文章模块 (article) ★ 重点 — AI 一键成文

本模块提供**两种创作方式**：

#### 方式一：手动编写

用户进入 Markdown 编辑器 → 输入内容 → 选择分类/标签 → 保存草稿或发布。
发布后触发 ArticlePublishEvent → 异步向量化入库。

#### 方式二：AI 一键成文（模糊想法 → 完整文章）★ 新增

根据当前编辑页面的内容，点击ai润色直接返回ai生成的内容

**后端实现要点：**

```java
// ArticleGenerateService — 核心类
public class ArticleGenerateService {
    
    // SSE 流式生成文章
    public Flux<String> generate(ArticleGenerateReq req, Long userId) {
        // 1. 组装 System Prompt
        String systemPrompt = buildSystemPrompt(req.getGenre(), req.getAudience(), req.getLength());
        
        // 2. 调用 AI 流式生成
        return aiChatService.chatStream(systemPrompt, req.getIdea());
    }
    
    // 发布 AI 生成的文章
    public Article publish(ArticleGenerateReq req, String content, Long userId) {
        Article article = new Article();
        article.setTitle(extractTitle(content));      // 从内容提取标题
        article.setContent(content);
        article.setSourceType("AI_GENERATED");
        article.setSourcePrompt(req.getIdea());        // 保存原始模糊输入
        article.setAuthorId(userId);
        article.setStatus("PUBLISHED");
        // ... 保存到数据库
    }
}
```

### 5.4 互动模块 (interaction)

- **评论：** 支持两层嵌套回复，作者可软删除
- **点赞：** toggle 操作，Redis Set 缓冲 + 定时刷 MySQL
- **收藏：** toggle 操作，用户个人中心查收藏列表
- **阅读量：** Redis INCR + 定时 5 分钟批量同步到 MySQL

### 5.5 AI 模块 (ai)

#### RAG 知识库问答

```
入库：文章发布 → @Async → Markdown 按 ## 分块(512tokens, overlap=64)
     → Embedding(text-embedding-3-small) → PGVectorStore 存储
     → chunk_hash 去重，幂等

查询：用户提问 → Embedding → similaritySearch(topK=5, threshold=0.7)
     → 组装 Prompt + 来源引用 → SSE 流式返回
```

#### AI 写作助手

| 功能 | 接口 | 说明 |
|---|---|---|
| 纠错 | POST /api/ai/writer/proofread | 修正拼写/语法/Markdown格式 |
| 润色 | POST /api/ai/writer/polish | 改善表达 |
| 翻译 | POST /api/ai/writer/translate | 中英互译 |
| 扩写 | POST /api/ai/writer/expand | 扩展段落 |
| 摘要 | POST /api/ai/writer/summarize | 生成 TL;DR |
| 标签推荐 | POST /api/ai/writer/recommend-tags | 推荐 3-5 个标签 |

#### BYOK

用户设置页填入自己的 API Key → AES-256-GCM 加密存储 → 调用时动态创建临时模型 → 用完销毁

### 5.6 积分模块 (points)

| 行为 | 积分 | 每日上限 | 实现 |
|---|---|---|---|
| 每日签到 | +10 | 1次 | Redis Bitmap |
| 阅读文章 | +1 | 10次 | Redis Set |
| 点赞 | +2 | 5次 | Redis 计数器 |
| 评论 | +3 | 10次 | Redis 计数器 |
| 兑换 AI 配额 | -100 | - | 100分=1次AI对话 |

### 5.7 排行模块 (ranking)

```
热榜公式：
score = view×1 + like×5 + comment×10 + bookmark×3
      × 时间衰减 (1/log(天数+1)+1)

Redis Sorted Set: ZADD hot:articles <score> <id>
查询: ZREVRANGE hot:articles 0 49 → Top 50
每 10 分钟定时更新
```

### 5.8 秒杀模块 (flashsale)

```
Lua 原子操作：
  ① 检查用户已购 < 限购
  ② 检查库存 > 0
  ③ DECR stock
  ④ SADD user_set
  ⑤ INCR sold_count
  ⑥ 返回 1 成功 / 0 库存不足 / -1 限购

Redisson 分布式锁(用户级串行) + 异步削峰创建订单
```

### 5.9 优惠券模块 (coupon)

Admin 创建模板 → 用户领取 → 兑换时抵扣积分 → 过期标记
