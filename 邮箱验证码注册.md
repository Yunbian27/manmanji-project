# 注册邮箱验证码功能

> 用户注册前必须先用邮箱接收验证码，验证通过后才能创建账号。

## 背景

原有注册仅校验用户名和邮箱是否重复，存在两个隐患：
- 恶意注册：可随意填写他人邮箱批量注册
- 邮箱不真实：用户打错邮箱后无法找回密码

增加验证码机制确保注册邮箱真实有效。

## 注册流程

```
用户输入邮箱 → 点"发送验证码" → 后端生成6位随机码 → 存 Redis（5min TTL）
→ 异步发 HTML 邮件 → 用户输入验证码 → 提交注册
→ 后端从 Redis 取码比对 → 一致则创建用户，删除 Redis 中的验证码
```

## 核心实现

### 发送验证码

```java
// AuthService.sendCode()
public void sendCode(String email) {
    // 限流：同一邮箱 60s 内不可重复发送
    String rateKey = RedisKeys.REGISTER_CODE_RATE_PREFIX + email;
    if (Boolean.TRUE.equals(stringRedisTemplate.hasKey(rateKey))) {
        throw new BusinessException(ErrorCode.CODE_SEND_TOO_FREQUENT);
    }

    // 生成 6 位数字验证码
    String code = String.format("%06d", ThreadLocalRandom.current().nextInt(1000000));

    // 存入 Redis，5 分钟过期
    stringRedisTemplate.opsForValue().set(
        RedisKeys.REGISTER_CODE_PREFIX + email, code, Duration.ofMinutes(5));
    // 限流标记，60 秒
    stringRedisTemplate.opsForValue().set(rateKey, "1", Duration.ofSeconds(60));

    // 异步发邮件
    mailService.sendCode(email, code);
}
```

### 验证码校验（register 方法中新增）

```java
String codeKey = RedisKeys.REGISTER_CODE_PREFIX + req.getEmail();
String storedCode = stringRedisTemplate.opsForValue().get(codeKey);
if (storedCode == null) {
    throw new BusinessException(ErrorCode.CODE_EXPIRED);      // 验证码过期
}
if (!storedCode.equals(req.getCode())) {
    throw new BusinessException(ErrorCode.CODE_INCORRECT);    // 验证码错误
}
stringRedisTemplate.delete(codeKey);  // 一次性使用，验证后删除
```

### 邮件 HTML 模板

使用 `MimeMessageHelper` 发送 HTML 格式邮件，设置 `helper.setText(html, true)`。邮件内容是简单的居中卡片样式，包含品牌名 + 验证码 + 安全提示。

### 前端倒计时

```typescript
// 点发送按钮 → 按钮置灰 60s，每秒递减
countdown.value = 60
const timer = setInterval(() => {
  countdown.value--
  if (countdown.value <= 0) clearInterval(timer)
}, 1000)
```

## 踩坑记录

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| `AuthenticationFailedException: 535` | 163/QQ 邮箱 SMTP 需要授权码，不是登录密码 | 去邮箱设置开启 SMTP，获取授权码填入 `MAIL_PASSWORD` |
| `can't determine local email address` | `MimeMessageHelper` 未设发件人 `from` | 添加 `helper.setFrom(mailFrom)`，`mailFrom` 通过 `@Value("${spring.mail.username}")` 注入 |
| `[EOF]` bad greeting | 端口 465 用了 STARTTLS 而非 SSL | 465 端口→`mail.smtp.ssl.enable: true`；587 端口→`mail.smtp.starttls.enable: true` |
| 多个 TaskExecutor 导致 `@Async` 不确定用哪个 | 项目中有 `asyncExecutor`、`aiExecutor`、`taskScheduler` | `@Async("asyncExecutor")` 明确指定 |

## 配置注意事项

```yaml
spring:
  mail:
    host: smtp.163.com          # 或 smtp.qq.com
    port: 587                    # 587=STARTTLS, 465=SSL
    username: ${MAIL_USERNAME}   # 环境变量，不提交 git
    password: ${MAIL_PASSWORD}   # 授权码，不是登录密码
    properties:
      mail.smtp.auth: true
      mail.smtp.starttls.enable: true   # 587 端口用这个
      # mail.smtp.ssl.enable: true       # 465 端口用这个
```
